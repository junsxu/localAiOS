## 记忆类型全景

```
                    Memory 系统
                         │
        ┌────────────────┼─────────────────┐
        │                │                 │
   短期记忆          长期记忆          知识外脑
 (Working Mem)    (Long-term)         (RAG)
        │                │
  当前对话历史      Qdrant 向量库
  当前任务状态      SQLite 用户画像
  临时计算结果      历史摘要/偏好
        │
        └──── 对话结束时摘要 ────→ 长期记忆
```

## 核心知识速览

### 短期 vs 长期记忆

| 维度 | 短期记忆 | 长期记忆 |
|------|----------|----------|
| 生命周期 | 单次会话 | 跨会话、永久 |
| 存储 | 内存（Python list） | Qdrant + SQLite |
| 读取方式 | 直接注入 Prompt | 向量检索后注入 |
| 内容类型 | 完整对话轮次 | 摘要、偏好、决策 |
| 消耗资源 | 模型上下文窗口 | 存储空间 + 检索时间 |

### 记忆读写触发时机

**写入触发**（满足任一）：
- 对话结束 → 自动摘要并存储
- 用户明确提到偏好："我喜欢简洁的代码风格"
- 重要决策："我们决定用 PostgreSQL 而不是 MySQL"
- 对话超过 20 轮 → 滑动摘要

**读取触发**：
- 每次新对话开始，用用户 ID 检索最近 K 条记忆
- 用当前问题做语义搜索，找相关历史

## 实操任务

### 任务 1：最小可运行 Memory 对话

实现带记忆的对话循环：对话过程中实时检索历史记忆注入 System Prompt，对话结束时自动调用 LLM 对本次对话做摘要，并将摘要存入 Qdrant，下次对话时可检索到。

> 完整实现（异步、多用户、缓存）见 [conversation.md](./conversation.md)。

- [ ] 对话结束保存记忆，下次对话能检索到历史

---

### 任务 2：用户画像提取

检测对话中的偏好关键词（"我喜欢"、"我偏好"、"以后都用"等），自动触发画像写入，将用户偏好持久化到 SQLite，后续对话的 System Prompt 中自动带入相关偏好。

> 完整画像建模和自动提取见 [user_profile.md](./user_profile.md)。

- [ ] 说出"我喜欢简洁的代码"，AI 在下次对话中能应用此偏好

---

### 任务 3：记忆注入 Prompt

设计 `build_system_prompt` 函数：根据用户 ID 和当前问题，同时检索向量记忆（语义相关历史）和用户画像（固定偏好），组合为结构化的 System Prompt 头部，让 AI 在每次回答时天然带入用户背景。

> 完整设计和 Prompt 模板见 [memory.md](./memory.md)。

- [ ] System Prompt 包含检索到的历史记忆，AI 能自然地引用
