# 贡献指南

感谢你对 **Local AI OS** 的关注！这是一份面向个人开发者的本地 AI 系统构建路径，内容来自实践踩坑，欢迎一起完善。

---

## 这个项目接受什么样的贡献

本项目的定位是**学习路径与踩坑记录**，不是通用部署工具。因此：

**欢迎**：
- 修正文档中的错误（命令、路径、版本号等）
- 补充特定环境下的踩坑记录（例如：不同 Ollama 版本的差异）
- 改善文档表述，使其更清晰易懂
- 补充缺失的说明或配置细节
- 报告文档与实际操作不符的问题

**不在维护范围内**：
- 新增 Web UI 或可视化界面
- AMD GPU / 纯 CPU 的适配分支
- 与本项目定位无关的新功能

## 提 Issue

### Bug 报告

文档中的命令执行报错，或步骤无法复现，请按以下格式提 Issue：

```
**所在阶段**：阶段 X（例如：阶段 2 · RAG）
**操作系统**：Windows 11 版本 / WSL Ubuntu 版本
**具体问题**：（描述你做了什么，期望发生什么，实际发生了什么）
**错误信息**：（完整的命令输出或错误日志）
**已尝试的方法**：（你排查过哪些方向）
```

### 提问 / 疑惑

对某个概念或步骤不理解，请按以下格式提 Issue：

```
**所在阶段**：阶段 X
**具体问题**：（你的问题是什么）
**已查阅的文档**：（指向具体文档章节的链接）
```

## 提交 Pull Request

1. **Fork** 本仓库，基于 `main` 分支创建新分支

   ```bash
   git checkout -b fix/stage2-qdrant-port
   ```

2. 修改对应的 Markdown 文件

3. 确认改动范围：
   - 修复错误：尽量只改有问题的那处，不要大范围重构
   - 补充内容：建议在文档末尾的"踩坑记录"或"常见问题"小节追加，保持原文结构

4. Commit 信息使用简短的中文或英文描述：

   ```
   fix: 修正 Nginx upstream 端口号（11435 而非 11434）
   docs: 补充 WSL Mirrored 模式在 Win10 下不可用的说明
   ```

5. 提 PR，在描述中说明：
   - 修改了哪个文档
   - 问题是什么（如果是 fix）
   - 在什么环境下验证过（如果涉及命令或配置）

## 文档规范

- 所有文档使用 **Markdown** 格式
- 代码块注明语言（`` ```bash ``、`` ```python ``、`` ```yaml `` 等）
- 命令行示例中，带有 `$` 前缀表示普通用户，`#` 前缀表示 root
- 涉及具体版本号时，请注明验证日期（例如：`# 验证于 2026-05，Qdrant v1.13`）
- 不要在文档里硬编码个人 IP 或私有域名，使用占位符（如 `<NAS_IP>`）

---

## 响应时间

这是个人项目，会在 **72 小时内**回复 Issue（即使只是"已收到，正在排查"）。  
如果 Issue 解决了文档问题，修复会在对应文档中补充说明，关闭 Issue 时会注明位置。
